# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add dark mode to the forum's user-facing UI using Tailwind CSS `dark:` variants, toggled by a sun/moon button in the navbar, defaulting to OS preference and persisting in `localStorage`.

**Architecture:** A synchronous inline `<script>` in `<head>` applies the `dark` class to `<html>` before first paint (preventing flash). A Stimulus controller attached to `<body>` handles user toggle and live OS preference changes. All color changes are Tailwind `dark:` utility classes.

**Tech Stack:** Tailwind CSS v4 (`tailwindcss-rails` gem), Stimulus (Hotwire), `localStorage`, `prefers-color-scheme` media query.

- [ ] **Step 4: Create `dark_mode_controller.js`**

Create `app/javascript/controllers/dark_mode_controller.js`:

```js
import { Controller } from "@hotwired/stimulus"

export default class extends Controller {
  connect() {
    if (!localStorage.getItem('theme')) {
      this._mediaQuery = window.matchMedia('(prefers-color-scheme: dark)')
      this._listener = (e) => {
        document.documentElement.classList.toggle('dark', e.matches)
      }
      this._mediaQuery.addEventListener('change', this._listener)
    }
  }

  disconnect() {
    if (this._mediaQuery && this._listener) {
      this._mediaQuery.removeEventListener('change', this._listener)
    }
  }

  toggle() {
    const isDark = document.documentElement.classList.toggle('dark')
    localStorage.setItem('theme', isDark ? 'dark' : 'light')
    if (this._mediaQuery && this._listener) {
      this._mediaQuery.removeEventListener('change', this._listener)
      this._mediaQuery = null
      this._listener = null
    }
  }
}
```
